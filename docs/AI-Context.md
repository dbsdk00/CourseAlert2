# 🧠 AI Handover Context & Memory

이 문서는 AI 에이전트(당신)가 CourseAlert 프로젝트의 새로운 세션을 시작할 때, 이전 대화의 문맥을 완벽하게 이어받고 프로젝트 아키텍처를 파악하기 위해 **가장 먼저 읽어야 하는 핵심 뇌(Brain) 저장소**입니다.

---

## 1. 🏗️ 프로젝트 정체성 및 아키텍처 요약
* **프로젝트명**: CourseAlert (성결대학교 수강신청 빈자리 PWA 알림 서비스)
* **아키텍처**: 완전한 **Zero-DB 구조** (별도의 백엔드 DB 없음)
  - 사용자의 알림 구독 목록(`alerts`)은 프론트엔드의 브라우저 `localStorage`에만 암호화/저장됩니다.
  - 백엔드(Node.js)는 오직 인메모리 세션 캐싱과 대학교 서버의 스크래핑(Scraping) 역할만 수행합니다.
* **보안 수준 (최고 등급)**:
  - 깃허브 공개(Public) 전환 완료. (`.env` 파일 철저히 git 추적 해제됨)
  - 암호화 키(VAPID)는 소스 코드에 하드코딩되지 않고 동적 페치(Dynamic Fetching)로 가져옵니다.

---

## 2. ✨ 최근 적용된 메이저 패치 내역 (히스토리)
새로운 작업을 진행할 때 아래 내용들이 이미 완료되었음을 인지해야 합니다.

1. **이중 중복 알림 이슈 완치**: 
   - 과거에는 백그라운드 PWA 푸시 알림과 브라우저 로컬 알림이 중복으로 격발되는 문제가 있었습니다.
   - **현재 상태**: 프론트엔드(`App.tsx`) 단의 로컬 알림 생성 로직을 완전히 주석 비활성화했습니다. 오직 클릭 시 수강신청 링크로 넘어가는 1개의 **백그라운드 공식 푸시(`push-sw.js`) 알림만 격발**되도록 튜닝 완료되었습니다.
2. **권한 허용 버튼 제거 및 가이드 모달 전면 도입**:
   - `RegisterForm.tsx` 하단의 `[알림 권한 허용하기]` 조건부 버튼을 **제거**했습니다.
   - 대신 메인 화면 상단에 `[3초 만에 알림 설정 완료하기]` 배너 버튼(`NotificationGuide.tsx`)을 신설하여, 텍스트가 아닌 실제 SVG 아이콘 기반의 고퀄리티 기종별 설정 가이드와 **[테스트 알림 보내기]** 버튼을 탑재했습니다.
   - **⚠️ iOS 특이사항 명시**: "애플 정책상 반드시 카카오톡 인앱 브라우저를 벗어나 **Safari**에서 **'홈 화면에 추가'**를 해야만 알림이 온다"는 치명적 제약 사항이 가이드에 추가되어 있습니다.

## 3. 🚨 현재 시스템의 한계점 및 다음 개발 과제 (Next Actions)
다음 에이전트가 코어 리팩토링을 지시받을 경우, 아래의 치명적인 아키텍처 한계점을 해결하는 방향으로 제안해야 합니다.

* **치명적 결함 (프론트엔드 폴링 의존성)**: 현재 시스템은 유저의 브라우저(`App.tsx`)가 켜져 있어야만 백엔드로 빈자리 탐색 API 요청(2초 폴링)을 보냅니다. 만약 유저가 휴대폰 화면을 끄거나 앱을 백그라운드로 넘기면(OS의 JS 중단), 아무도 대학교 서버를 감시하지 않게 되어 백그라운드 알림이 100% 누락되는 구조적 결함이 잠재해 있습니다.
* **미래의 목표 (Event-Driven 데몬 전환)**:
  - 프론트엔드의 `setInterval` 폴링을 전면 폐기해야 합니다.
  - 백엔드 서버 혼자서 24시간 독립적으로 봇(Cron Daemon)을 돌려 학교 서버를 크롤링하고, 빈자리가 감지된 순간 해당 과목을 구독 중인 VAPID 토큰들을 모아 한 번에 브로드캐스트 푸시를 날리는 형태로 아키텍처 대수술이 필요합니다.

## 4. 🎨 UI/UX 디자인 시스템 제약 사항
* **색상 팔레트**: `index.css`의 토큰을 철저히 준수하세요.
  - `--bg-1` (#161616), `--bg-2` (#1e1e1e), `--bg-3` (#2a2a2a)
  - `--accent` (#d4af37 - 골드), `--green` (#34d399 - 성공), `--red` (#ef4444 - 경고)
* **스크롤바 및 오버스크롤**: 
  - 모바일 기기에서의 탄성 스크롤(Rubber-band) 시 이질감을 막기 위해 HTML/BODY 배경색이 `#141414`로 통일되어 있으며, 검색 결과 창에만 얇은 커스텀 스크롤바(5px)가 적용되어 있습니다.
* **No Unused Variables (엄격한 빌드 규칙)**: 
  - `Vite / tsc` 빌드 룰이 매우 엄격합니다. 한 개의 미사용 변수(예: TS6133)라도 발생하면 Vercel 배포가 중단되므로, 사용하지 않는 import나 변수는 완벽히 청소하세요.
